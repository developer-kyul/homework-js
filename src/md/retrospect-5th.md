# JS 5주차 과제 - 요금제 플랜 컴포넌트

## 📌 과제 수행 과정 기록

이번 과제는 단순히 “카드를 클릭하면 선택된다”는 기능 구현보다, 선택 상태를 어떻게 관리할 것인가를 깊이 고민하게 만든 과제였다.
처음에는 클릭 이벤트 자체는 쉽게 구현했지만, 모든 카드가 동시에 선택되거나, 다시 클릭했을 때 선택이 해제되지 않는 문제를 겪으며 로직을 여러 번 수정하게 되었다.

특히 classList.toggle()을 무작정 사용했을 때 발생하는 문제를 직접 경험하면서,
“어떤 요소를 기준으로 상태를 변경해야 하는가”가 UI 로직에서 얼마나 중요한지 체감할 수 있었다.
이후 e.target.closest()를 활용해 실제로 클릭된 카드 하나만 정확히 제어하도록 구조를 바꾸면서, 코드가 훨씬 명확해졌다.

또한 카드에 data-plan, data-price와 같은 data 속성을 정의하고, 이를 통해 선택된 플랜 정보를 하단 안내 영역에 출력하는 흐름을 구현했다.
이 과정에서 dataset 사용법, 선택자 오타로 인한 null 에러, querySelector와 querySelectorAll의 차이를 직접 디버깅하며 이해하게 되었다.

## 📌 과제 코드 리뷰

HTML과 CSS가 이미 준비된 상태에서 작업을 진행했다. 물론 처음부터 직접 구현하는 것도 가능했지만, 잘 정리된 코드를 먼저 접하고 그 구조를 읽어보는 경험이 나에게 더 의미 있는 시간이라고 생각했기 때문이다. 마크업이 명확하게 역할별로 나뉘어 있어, JavaScript를 어디에 어떻게 연결해야 할지 자연스럽게 떠올릴 수 있었고, 코드 흐름을 이해하는 데에도 많은 도움이 되었다. 이 과정을 통해 단순히 동작하는 코드가 아니라, 개발자 입장에서 읽기 쉽고 다루기 편한 코드가 무엇인지를 실제로 체감할 수 있었다.

## 📌 회고 (깨달은 점, 아쉬운 점 등)

이번 과제를 통해 UI 구현의 핵심은 이벤트 자체가 아니라 상태 관리라는 점을 분명히 느꼈다.
한 번 더 클릭하면 선택이 취소되는 로직을 구현하면서,
“현재 상태 → 다음 상태”를 명확히 구분하지 않으면 코드가 쉽게 꼬인다는 것을 실감했다.

또한 디버깅 과정에서 콘솔 로그를 적극적으로 활용하며
null 에러의 원인을 추적하고, 선택자와 DOM 구조를 다시 확인하는 습관이 많이 자리 잡았다.
AI를 통해 내가 작성한 코드를 기반으로 피드백을 받으면서,
단순히 정답 코드를 보는 것이 아니라 왜 이 코드가 더 나은지를 이해할 수 있었던 점도 큰 수확이었다.

아쉬운 점이 있다면, 처음부터 상태 흐름을 충분히 설계하지 않고 구현부터 시작했다는 점이다.
하지만 그 시행착오 덕분에, 확실하게 배운 점이 많다.
