# JS 4주차 과제 - 카드 탭

## 📌 과제 수행 과정 기록

드디어 과제 레포지토리를 정리했다. 그동안 작업은 계속해왔지만, 체계적으로 쌓아가고 있다는 느낌이 부족해 고민이 많았다. 여러 번 구조를 고민하고 시간을 들인 끝에, 이제야 조금 정리가 되었다는 안정감이 든다.
이렇게 시간이 오래 걸린 것은, 과정 자체를 기록하고 나중에 포트폴리오로 활용할 수 싶다는 욕심이 더 커졌던 것 같다. 그만큼 속도는 더뎠지만, 지금 돌아보면 그 고민의 시간도 충분히 의미 있었다고 느낀다.

## 📌 과제 코드 리뷰

HTML과 CSS 코드를 미리 제공받아 작업할 수 있었던 점이 정말 좋았다. 단순히 구현하는 과제가 아니라, 실제 프론트엔드 개발자가 되어 잘 정리된 코드를 분석하고 이해하는 경험을 한 기분이었다. 특히 구조가 명확한 HTML과 완성도 높은 CSS 코드를 직접 보며, “좋은 코드”가 무엇인지 체감할 수 있었던 시간이 인상 깊었다.
개발 실력은 결국 다양한 코드를 접하고 분석하면서 자연스럽게 쌓인다고 느끼는데, 이번 과제에서는 그 과정을 충분히 경험할 수 있었다. 그래서 구현보다 코드 분석에 더 많은 시간을 썼고, 그 시간이 매우 만족스러웠다.

## 📌 회고 (깨달은 점, 아쉬운 점 등)

이번 과제를 통해 다시 한 번 “개발은 환경 설정과 설계가 반이다”라는 말을 실감하고 있다. 이미 몇 번이나 구조를 뒤집고 수정했는지 모를 정도로 시행착오가 많았다. 지금 상태가 과제 레포지토리의 최종 버전이지만, 이후 리액트로 넘어가면 개발 환경이 또 달라질 것을 알기에 이 경험 자체가 더욱 중요하게 느껴진다. 그럼에도 불구하고, 현재 JS 환경 설정은 어느 정도 기준을 잡았다는 점에서 큰 성취감을 느낀다.

또한 콘솔 로그를 적극적으로 활용하며 데이터 흐름을 확인했고, 내가 작성한 코드의 부족한 점을 AI를 통해 첨삭받았다. 직접 작성한 코드를 기반으로 피드백을 받으니, 어떤 부분이 약한지 명확하게 보였고 코드를 바라보는 시야도 한층 넓어졌다. 실무에서는 기존 코드를 가져다 사용하는 경우가 많지만, 자주 쓰이는 로직을 직접 구현해볼 수 있었다는 점이 특히 의미 있었다.
