# JS 3주차 과제 - 게임 슬라이드

## 📌 과제 수행 과정 기록

이번 과제를 진행하면서 단순히 과제를 끝내는 데 그치지 않고, 이후 포트폴리오에서도 활용하고 싶다는 욕심이 생겨 여러 가지 작업을 동시에 벌이게 되었다. 그 영향인지 최근에는 과제 수행과 학습 노트 정리를 병행하는 과정에서 정신적으로 여유가 없는 상태가 계속되고 있다.

기존에는 학습 내용을 단락별로 정리하는 방식을 선호했는데, 이 방법이 구조적으로는 정리가 잘 되지만 생각보다 시간이 많이 소요된다는 단점이 있었다. 그래서 날짜별 정리 방식으로 바꿀지 고민했지만, 실제로 다시 찾아보거나 복습할 때는 단락별 정리가 훨씬 효율적이라는 점을 느껴 쉽게 방향을 바꾸지 못했다.

또한 실행 환경 설정과 배포 과정까지 직접 다루다 보니 프로젝트와 레포지토리가 빠르게 늘어나 관리 부담도 커졌다. 정리해야 할 것들이 끝없이 쌓여가는 느낌 속에서, 작업이 쉽게 끝나지 않는다는 피로감과 함께 과제를 수행하고 있는 요즘이다.

## 📌 과제 코드 리뷰

과제 코드를 다시 돌아보며 가장 먼저 느낀 점은 CSS 구조가 상당히 정리되지 않았다는 것이었다. 클래스 네이밍을 처음에는 일정한 규칙으로 작성하다가, 작업이 진행될수록 \_\_ 방식과 - 방식을 혼용하게 되어 전체적인 일관성이 무너졌다.

변수 네이밍 역시 명확한 기준 없이 즉흥적으로 작성된 부분이 많아, 시간이 지나 다시 코드를 읽을 때 의도를 파악하는 데 불필요한 노력이 들 것 같다는 생각이 들었다. 결과물을 완성한 뒤에야 이러한 문제들이 더 크게 보였고, 실제 프로젝트를 시작할 때는 반드시 초반 단계에서 네이밍 규칙과 구조를 명확히 정해야 한다는 중요성을 실감했다.

## 📌 회고 (깨달은 점, 아쉬운 점 등)

이번 과제를 돌아보면 아쉬운 점이 상당히 많다. 특히 작업 중간중간 Git에 커밋을 남기지 못해, 개발 과정을 되돌아보거나 흐름을 정리하기가 어려웠다. 결과물만 보면 크게 문제는 없지만, 학습 과정이라는 점을 생각하면 기록을 남기지 못한 부분이 가장 아쉽게 느껴진다.

물론 실무에서는 결과가 중요한 경우가 많지만, 지금은 배우는 단계이기 때문에 시행착오와 고민의 흔적 자체가 큰 자산이 된다고 생각한다. 이번 경험을 통해 앞으로는 작업 단위가 작더라도 자주 커밋하고, 과정 중심의 기록을 남기는 습관을 들여야겠다고 다짐하게 되었다.

완성도도 중요하지만, 성장 과정을 스스로 증명할 수 있는 기록의 힘을 다시 한 번 느낀 과제였다.
